Arch安装教程（UEFI）

Arch是什么:

Arch Linux（或Arch /ˈɑːrtʃ/)）是一款基于 x86-64 架构的 Linux 发行版。系统主要由自由和开源软体组成，支持社区参与。系统设计以KISS原则（Keep It Simple and Silly,保持简单和愚蠢）为总体指导原则，注重代码正确、优雅和极简主义，期待用户能够愿意去理解系统的操作。 Arch Linux 系统安装、删除和更新软体的软体包管理器叫做 Pacman

ArchLinux的优势

  1. AUR(ArchLinux User Repo)中完善的社区软件包
  2. 对于中国大陆用户友好的ArchLinux CN源
  3. 滚动更新发行版，没有跨版本更新失败的苦恼
  4. 基本上完全自定义，想要什么就自己安装什么，并没有其他发行版安装一堆自己不需要的东西的苦恼
  5. 虽然是命令行安装，但是也算是较为友好且直观了，并且没有安装向导就不会出现Debian安装向导那种莫名其妙的报错

ArchLinux的问题

  1. 滚动更新发行版较为不稳定（虽然还是比隔壁Windows稳定）（除非你有十足的把握，请不要将其用于服务器）
  2. AUR内的软件包有很大一部分引用了GitHub内的资源，导致对于中国大陆并不是很友善
  3. 滚动更新时对于nVIDIA显卡驱动的处理很容易出错，导致无法正常进入系统

接下来就是正式安装

  • 接下来的内容全程都是在Live-CD的root账户或chroot环境的root账户下进行的，您并不需要注意当前用户

  1. 下载安装盘 这里推荐使用国内镜像站 (https://mirrors.bfsu.edu.cn/archlinux/iso) 建议选择最新版本进行安装

  2. 刻录安装盘至U盘
    网络上关于如何刻录启动盘的教程很多，不进行赘述
    推荐Rufus

  3. 从U盘启动，引导加载器这里我们选择第一个

  4. 连接到网络

    • 确保系统已经启用了网络接口，用ip-link检查：
    • 要连接到网络：
      • 有线以太网 —— 连接网线
      • WiFi —— 使用iwctl验证无线网络
        • 首先进入iwctl的交互式界面

    timedatectl set-ntp true


    cfdisk

    lsblk

    mkfs.ext4 /dev/root_partition（根分区）

    mkfs.fat -F32 /dev/efi_partition（efi分区）

    mkswap /dev/swap_partition（交换空间分区）

    mount /dev/root_partition（根分区） /mnt

    mkdir /mnt/efi

    mount /dev/efi_partition (efi分区) /mnt/efi

    swapon /dev/swap_partition（交换空间分区）

    curl -L -o /etc/pacman.d/mirrorlist “https://www.archlinux.org/mirrorlist/?country=CN“

    pacstrap /mnt base linux linux-firmware vim base-devel

    genfstab -U /mnt >> /mnt/etc/fstab

    genfstab -U /mnt >> /mnt/etc/fstab

    127.0.0.1 localhost
    ::1 localhost

    passwd

    pacman -S dosfstools grub efibootmgr

    grub-install –recheck /dev/sda –efi-directory=/efi

    grub-mkconfig -o /boot/grub/grub.cfg

    useradd -m -G wheel -s /bin/bash uesrname（用户名可以改成你喜欢的）

    passwd username（同上方用户名）

    pacman -S dhcp dhcpcd

    systemctl enable dhcpcd

    exit

    reboot

    ```

    • 如果是使用WiFi进行联网的，重启之后第一次启动还需要再次连接WiFi，方法见上文
